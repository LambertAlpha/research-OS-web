# Folder: src

1. **地位**: 应用根源码目录。所有前端业务代码的顶级容器，组织页面路由、UI 组件、服务层和类型契约。
2. **逻辑**: 通过 Next.js App Router 约定式路由组织页面；组件层纯渲染、服务层管理 API 通信、类型层约束数据契约。
3. **约束**: 所有 import 必须使用 `@/` 别名；严禁在 `src/` 外放置业务逻辑代码。

## 成员清单

- `app/`: 页面路由层 — Next.js App Router 页面组件与全局布局 (Route Definitions)
- `components/`: UI 组件层 — 可复用的 React 展示组件 (Presentational Components)
- `lib/`: 服务层 — API 客户端与工具函数 (Service & Utilities)
- `types/`: 契约层 — TypeScript 类型定义，前后端数据结构的唯一真相源 (Type Contracts)

**触发器**: 一旦本文件夹增删子目录或架构逻辑调整，必须重写此文档。
